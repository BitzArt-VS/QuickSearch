# Component Model

`IGuiComponent` is the layout-participating framework contract. It extends `IGuiNode` and `IGuiMeasurable`. `GuiComponent` is the default base class. Custom components may implement `IGuiComponent` directly.

**`IGuiComponent` members:**
- `LayoutParameters` — size/margin/padding/positioning, consumed by layout pass.
- `RenderFragment` — stable delegate wrapping `BuildRenderTree`. Identity must not change.
- `Attach(handle)` — called once by framework after construction.
- `OnInitialized()` — first mount only.
- `OnParametersSet()` — every pass.
- `Render(context, bounds)` — dirty-paint draw hook, before children, during the current combined arrange/paint walk.
- `RenderOverlay(context, bounds)` — dirty-paint draw hook, after this component's own children, before the next sibling slot.
- `Measure(availW, availH)` — content-size hook for `FitContent`. The base `GuiComponent` default delegates to `GuiComponentLayout.MeasureContent(RenderHandle.Slot.Children, ...)`. Leaf components override it to report intrinsic size. Custom container components may override it to replace or transform child measurement; call `base.Measure(availW, availH)` or use `GuiComponentLayout.MeasureContent(...)` directly to reuse the default stack measurement.

**`IGuiMeasurable`:** small capability interface containing only `Measure(availW, availH)`. It allows APIs such as resize to depend on measurability without also requiring layout participation.

**`IGuiResizable`:** capability interface for objects that can accept framework-suggested resize bounds. It extends `IGuiMeasurable`, not `IGuiComponent`, and exposes `SupportedResizeEdges` plus `Resize(GuiComponentBounds)`. The framework automatically records resize regions for arranged slots implementing this interface and calls `Resize` with proposed outer bounds; the target applies any component-specific limits internally.

**Configuration lifecycle:** Configuration actions registered during a blueprint pass apply only to that pass. At the start of each pass, when `AddComponent<T>(key)` reaches an existing slot, the slot's frame is `Reset()` — its accumulated action chain is cleared. The user's `BuildRenderTree` then re-registers actions for the current pass via `.Configure(...)` / `.ConfigureLayout(...)` / inline named arguments. During patch, layout components first reset `LayoutParameters` to framework defaults, then `GuiNode.ConfigureSlot(IGuiSlotBuilder)` runs, then external configuration actions run on the live instance.

This means **per-pass values take effect immediately** — passing `width: 200` to `Add<T>` next pass will resize the slot. There is no first-pass-wins trap. Closures still work the same way (a `.Configure(c => c.X = field)` call reads the latest field value because the action is freshly registered each pass).

**Own-slot configuration:** `GuiNode` subclasses can override `ConfigureSlot(IGuiSlotBuilder)` to export component-owned slot configuration after the instance exists but before external configuration applies. Use it for component layout defaults and for mouse/key callbacks whose target is the component's own allocated bounds. External render-tree configuration wins over layout defaults. Event callbacks are additive: own-slot callbacks run first, then callbacks registered by `.OnMouse*` / `.OnKey*` on the external builder. The same slot extension methods work in both contexts.

**Public API (extension methods):**
- Slot declaration (`RenderTreeBuilderExtensions`): `Add<T>(key, ...)`, `AddContainer(key, ..., background, content)`, `AddContainer<T>(key, ..., content)` (generic for `GuiContainer` subtypes), `AddLabel(key, text, font, ...)`. Component declarations accept layout as inline named parameters: `width`, `height`, `widthMode`, `heightMode`, `fill`, `margin`, `padding`, `direction`, `positioning`. `fill: true` is shorthand for setting both modes to `Fill`. Pure `IGuiNode` declarations are layout-transparent, so do not pass layout arguments to them.
- Per-slot config (`ComponentBuilderExtensions`): `.Configure(Action<T>)` for arbitrary state, `.ConfigureLayout(Action<GuiComponentLayoutParameters>)` for layout state.
- Component-owned slot config (`GuiNode.ConfigureSlot`): receives `IGuiSlotBuilder`, so own-slot defaults use the same `ConfigureLayout`, `OnMouse*`, and `OnKey*` extension methods as external slots.
- `WithContent(fragment)` on `IGuiComponentBuilder<GuiContainer>` for setting `Content` after the fact (prefer the inline `content:` argument).
- `RequestReconcile()` — protected on `GuiNode`; schedules a scoped rebuild of this node's render fragment and cascades to arrange + paint.
- `RequestArrange()` — protected on `GuiComponent`; recomputes layout/input regions for the existing component tree and cascades to paint. Use when bounds, scroll offsets, clipping, or input regions may have changed. It does not re-run `BuildRenderTree`; if layout declarations depend on changed state, use `RequestReconcile()`.
- `RequestPaint()` — protected on `GuiComponent`; redraws the latest arranged tree without requesting reconciliation. Use for visual-only state such as hover, pressed, focus, caret blink, or fixed-size value changes. `RequestRender()` remains as a compatibility alias.
- `IGuiRenderHandle` exposes the mounted `Slot` for every handle, including the root component handle, plus the same invalidation levels for direct implementors: `RequestReconcile(fragment)`, `RequestArrange()`, and `RequestPaint()`. Direct implementors must store the handle from `Attach()` before calling them.
- `GuiComponentLayout` exposes the same reusable measurement helpers used by the framework base class: `MeasureContent(...)` for default child-slot stack measurement and `ResolveAllocatedSize(...)` for fixed/fraction/fill/fit size resolution.

`IGuiRenderTreeBuilder.AddComponent<T>`, `IGuiComponentBuilder<T>.AddConfigurationAction`, and `IGuiSlotBuilder.Add*Handler` are `internal`; public registration goes through the `Add*` extension methods, configuration goes through `Configure`/`ConfigureLayout`, and event callbacks go through `OnMouse*` / `OnKey*`.

**Cascading parameters:** Blazor-style ambient values published to a subtree.

- **Producer:** `builder.AddCascadingValue<T>(value, content, name?)` — a **scope operation**, not a component. No key, no layout parameters. Implemented as `PushCascadeScope` on `IGuiRenderTreeBuilder`: saves the current `CascadeChain`, pushes a new link with `(typeof(T), name, value)` boxed as `object?`, invokes `content`, then restores. No slot is allocated; the layout tree is unaffected.
- **Consumer (GuiComponent helpers):** `protected T? GetCascadingValue<T>([string? name])`, `protected bool TryGetCascadingValue<T>([string? name,] out T value)`. Typically called in `OnParametersSet` to snapshot the value into a field.
- **Consumer (raw `IGuiComponent` implementors):** `IGuiRenderHandle.TryGetCascadingValue<T>([string? name,] out T value)`.
- **Resolution:** `(Type, Name)` exact match. Inner scopes shadow outer scopes with the same key. `null` name only matches `null` name. Walks innermost → root, returns first match.
- **Reactivity:** any ancestor reconcile recursively reconciles all descendants — consumers re-run `OnParametersSet` and naturally see the latest value. No subscriptions.
- **Internals:** `GuiRenderTreeBuilder` carries `CascadeChain` (the current blueprint scope for child declarations) and `InheritedCascadeChain` (the stable scope visible to the component that owns the builder). Both are immutable linked lists of `(Type, string?, object?)` entries. `PushCascadeScope<T>` mutates `CascadeChain` during the blueprint phase as a stack push/pop. `AddComponent<T>` snapshots `CascadeChain` onto the frame (`frame.CascadeChain = CascadeChain`) immediately after adding it to the frame list — so the snapshot is taken at the slot's lexical declaration site. During the patch phase, `slot.ChildBuilder.InheritedCascadeChain` and `slot.ChildBuilder.CascadeChain` are assigned from `frame.CascadeChain` before component callbacks run. `RenderHandle` reads `InheritedCascadeChain` on each lookup, so the component itself sees its declaration-site cascade scope without accidentally seeing temporary descendant scopes pushed while building its own child tree. Descendants see the same inherited scope, plus any scopes declared around them. `TryGet<T>` unboxes `object?` on match (one allocation for value types per lookup).

**`IGuiDialog` / `GuiDialog`:** `IGuiDialog` is the renderer-facing dialog contract: it extends `IGuiComponent` with offsets and optional runtime attachment only. Dialog-level keyboard and focus behavior uses the same slot callback system as normal components: the root dialog slot can register `OnKeyDown/Up/Press` and `OnFocusChanged` callbacks, and `GuiInputRouter` dispatches root-slot key callbacks before focused child callbacks. `GuiDialog` is the default base implementation; it keeps shared runtime attachment, `RequestClose`/`RequestFocus` helpers, `IsFocused` tracking, and implements `IGuiResizable`. Dialog-specific key/focus behavior belongs in normal `ConfigureSlot` subscriptions on the concrete dialog. Default Escape behavior is framework-owned by `GuiInputRouter`: blur the focused component first, then close the dialog. Dialog resize gesture mechanics live in internal `GuiResizeController` owned by the input router; `GuiDialog.Resize` only applies suggested screen bounds by clamping size, updating `OffsetX`/`OffsetY`, and calling `OnResizeUpdated(sizeChanged)`, while protected `ResizeEdges` lets subclasses choose edge sets without bypassing the `IsResizable` gate. The renderer and host are constrained to `class, IGuiDialog, new()`, not to `GuiDialog`, so other dialog implementations can exist. The static default `GuiDialogHost` owns normal application dialogs, while `new GuiDialogHostInstance(clientApi)` creates explicit isolated hosts when needed. Each host owns dialog renderer entries, and each renderer declares its dialog as a normal keyed component slot. Opening is renderer creation; closing removes the renderer entry. Dialog cleanup is opt-in: implement `IDisposable` on the concrete dialog (or any child component) only when it owns resources, and normal slot disposal will call it. Set default dialog dimensions through `ConfigureSlot(IGuiSlotBuilder)` so dialog reconciliation can reset and reapply layout like any other component.

**Dialog lifecycle notes:**
- For one-time setup, override `OnInitialized` on the dialog component itself — it runs once during the first reconcile before the component's child `BuildRenderTree` fragment is rebuilt.
- Dialogs do not expose `Open`/`Close` or an `IsOpen` state. Title bars, Escape handlers, and buttons request closure through the protected `RequestClose()` helper; the host decides by removing the dialog renderer entry.
